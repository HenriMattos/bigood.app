# Bigood Product Decisions

## Access Model

Bigood does not use public self-registration for barbershop owners.

The platform is accessed only through login after internal approval/configuration.

## Institutional Site

The institutional site is not a pricing page.

It must not show public plan prices.

It must not promote public signup or direct checkout.

Main CTA:

- Agendar demonstração

Secondary CTAs:

- Falar com especialista
- Ver como funciona
- Entrar

## Sales Model

Bigood uses consultative B2B sales.

The goal of the landing page is qualified lead generation.

The lead form should collect:

- name
- WhatsApp
- email
- barbershop name
- city/state
- approximate monthly revenue
- whether the barbershop works with subscription plans
- number of professionals
- number of units
- optional message

## Product Positioning

Bigood is a management system for barbershops, especially those that use or want to use customer subscription plans.

Core value:

- agenda organization
- client management
- recurring plan control
- service control
- professional management
- cash register/comandas
- finance
- customer portal

## Visual Identity

The current Bigood visual identity is approved.

Engineering work must preserve the visual system unless explicitly asked otherwise.
